# Handoff Checklist

Use this skill before final output.

## Verify

1. The requested outcome is implemented
2. Validation was run or explicitly skipped
3. User-facing behavior changes are mentioned
4. Risks or unknowns are clearly called out
5. File references point to the real edited files

## Do not

- hide failed validation
- imply testing that did not happen
- pad the summary with changelog noise
