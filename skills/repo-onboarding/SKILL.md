# Repo Onboarding

Use this skill when entering an unfamiliar codebase.

## Procedure

1. Read the root `README`
2. Read the root `AGENTS.md` if present
3. Identify package manager and test commands
4. Map the main source, config, and test directories
5. Summarize the architecture before editing

## Output

- repo purpose
- main languages and tooling
- likely edit points
- likely validation commands
