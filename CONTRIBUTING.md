# Contributing

Keep contributions small and composable.

## Preferred additions

- new Codex-oriented skills
- reusable agent role briefs
- rules with clear scope
- install and validation improvements

## Avoid

- giant monolithic prompts
- overlapping rules with conflicting guidance
- harness-specific assumptions without clear labels

## Contribution shape

1. explain the problem
2. add the smallest useful surface
3. update `README.md` if discoverability changes
4. update `manifests/components.json` if install surface changes
